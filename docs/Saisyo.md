# 指示
あなたは優秀なフルスタックエンジニアです。以下の要件と技術スタックに基づき、ブラウザ上で動作する高性能なPDF編集Webアプリケーションを開発してください。

# プロジェクト概要
ユーザーがPDFファイルをアップロードし、ブラウザ上で安全かつ直感的に編集・加工ができるSaaS型のWebアプリケーションを構築します。

# 技術スタック
- **フロントエンド / バックエンド:** Next.js (App Router)
- **スタイリング:** Tailwind CSS + shadcn/ui
- **データベース:** Turso (SQLite) + Drizzle ORM
- **認証:** Better Auth
- **ホスティング:** Vercel
- **ファイルストレージ:** Vercel Blob (PDFファイルの一時保存・管理用)
- **PDF処理ライブラリ:** `pdf-lib` (結合・分割・編集用), `react-pdf` またはブラウザネイティブのビューア (プレビュー用)

# コア機能（実装要件）
以下の機能を実装してください。PDFの処理はサーバーの負荷を減らすため、可能な限りクライアントサイド（ブラウザ上）の `pdf-lib` で実行する設計にしてください。

## 1. 認証・ユーザー管理 (Better Auth + Turso)
- メールアドレス・パスワードでのサインアップ/ログイン
- Google OAuthログイン（オプションとして準備）
- ユーザーのダッシュボード（過去の編集履歴や処理したファイルのメタデータ一覧を表示）

## 2. 基本的なPDF編集機能 (クライアントサイド処理)
- **PDFの結合 (Merge):** 複数のPDFファイルをアップロードし、ドラッグ＆ドロップで順序を入れ替えて1つのファイルに結合する機能。
- **PDFの分割 (Split):** 特定のページ範囲を指定して、PDFを抽出・分割する機能。
- **ページの削除・回転:** プレビューを見ながら、不要なページの削除や、特定のページの回転（90度ずつ）を行う機能。

## 3. 便利な拡張機能
- **パスワード保護の設定/解除:** PDFにパスワードをかける、またはパスワードが分かっているPDFの保護を解除する機能。
- **ウォーターマーク（透かし）の追加:** 任意のテキストをPDFの全ページに透かしとして追加する機能。

## 4. UI/UX要件
- ドラッグ＆ドロップでのファイルアップロード領域（Dropzone）。
- 処理中のローディング状態（スピナーやプログレスバー）の明確な表示。
- エラーハンドリング（ファイルサイズ上限の超過、破損したPDFのアップロード時のトースト通知）。
- モバイル対応（レスポンシブデザイン）。

# データモデル (Drizzle ORM)
Tursoに保存するテーブル構造の案です。必要に応じて拡張してください。
- `users`: Better Authの標準スキーマ
- `documents`: ユーザーがアップロード/編集したPDFの履歴（id, user_id, filename, file_url, operation_type, created_at）

# 開発ステップ
以下のステップに分けて、コードの生成と説明を行ってください。
1. **環境構築とDBスキーマ設定:** Next.jsのセットアップ、Drizzle ORMとTursoの接続、Better Authの設定。
2. **UIコンポーネントの実装:** shadcn/uiを用いたレイアウト、ヘッダー、ダッシュボードの作成。
3. **PDF処理ロジックの実装:** `pdf-lib` を用いた結合・分割・回転などのユーティリティ関数の作成。
4. **フロントエンド統合:** ファイルアップロードからPDF処理、ダウンロードまでのフローを繋ぎこむ。

まずはステップ1の「環境構築とDBスキーマ設定」から、必要なパッケージのインストールコマンドと初期設定コードを出力してください。
