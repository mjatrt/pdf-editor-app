npmワーム対策プロンプト

あなたはセキュリティエンジニアとして、現在のプロジェクト内に「Shai-Hulud（またはSha1-Hulud 2.0）」npmワームの痕跡がないか調査し、除去と修復を行ってください。以下のステップで実行し、各ステップの結果を報告してください。

1. IOC（侵害指標）のコードスキャン
以下の不審なファイルやスクリプトがプロジェクト内に存在しないか検索してください。

package.json 内の不審な preinstall スクリプト（setup_bun.js などを呼び出すもの）

意図しない setup_bun.js や、難読化された bundle.js の存在

.github/workflows/ 内の不審なファイル（shai-hulud-workflow.yml, discussion.yaml など）

2. 依存関係（npmパッケージ）の監査

package.json およびロックファイル（package-lock.json / yarn.lock / pnpm-lock.yaml など）をスキャンし、Shai-Huludの被害が報告されているパッケージの不審なバージョンアップや、依存ツリー内の異常を特定してください。

【修復指示】 もし侵害されたパッケージを発見した場合は、直ちに package.json から削除し、安全性が確認されているバージョンに固定して書き換えてください。

3. クリーンアップ用コマンドの生成
感染の疑いがある、あるいは予防措置として、プロジェクトをクリーンな状態に戻すためのターミナルコマンド（node_modules の削除、パッケージマネージャーのキャッシュクリア、再インストール）を出力してください。

4. 流出リスクのあるシークレットのリストアップ
Shai-Huludは環境変数から認証情報を盗み出すワームです。プロジェクト内の .env ファイル等の構成を確認し、今回ローテーション（再発行）が必要となる可能性のあるAPIキーやトークンのリストを洗い出してください。
